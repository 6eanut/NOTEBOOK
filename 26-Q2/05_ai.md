# AI: Prompt, Skill, Tool, Plugin, MCP, Agent

## Prompt

给LLM发的信息被称为Prompt。

## Skill

Prompt是一次性指令，Skill是可复用技能。

Skill可以理解为Prompt+Tool(可选)。

Skill可以被理解为一个workflow。

## Tool

Tool是可以被调用的外部功能，是可以操作的工具。

Skill和Prompt可以理解为“动嘴皮子”，Tool指的是“动手干活”。

## Plugin

Plugin可以理解为Tool的集合加上一个Tool的使用说明书。

说明书可以包含接口描述、参数规范、权限配置等。

## MCP

MCP是一个协议，不是一个实现。

它约定了Agent和Tool之间的通信协议和接口规范。

分为Client和Server两部分，Agent这边装Client，外部工具装Server。

双方用统一的协议通信，避免了每个外部工具需要给每个Agent做一套实现。

## Agent

早期大家直接用Prompt做一切，很快发现不可复用、不可维护。然后封装出Skill，发现还是只能动嘴皮子不能动手。于是引入Tool，发现Tool多了管理混乱。于是定义Plugin规范，发现每个平台规范不统一。于是推出MCP协议，最后发现没有大脑统筹，一切都是散兵游勇。于是Agent应运而生。每一层的出现，都是为了解决上一层解决不了的痛点。

Prompt是你对厨师说的一句指令，「我想吃糖醋里脊」。Skill是厨师练出来的手艺，做酸菜鱼的流程、做糖醋丸子的流程、做鸡蛋汤的流程，每样都是标准动作，不用你每次都教。Tool是锅铲烤箱秤，真实干活的物理工具。Plugin是工具的使用说明书，告诉你这个锅怎么开火、那个秤怎么读数。MCP是工具和主厨之间的通信协议，这个比较抽象不太好类比。Agent是主厨，统筹点菜、排菜、流程、把控最终结果。
